Steps:
1. Create multiple directories in "data" directory of mongoDB, such that each directory would represent a single instance of mongoDB.
	mkdir replicaset0
	mkdir replicaset1
	mkdir replicaset2

2. Start all mongoDB instances(each on different terminals)
	mongod --port 27017 --dbpath "C:\DevTools\MongoDB\Server\4.2\data\replicaset0" --replSet myreplicaset
	mongod --port 27019 --dbpath "C:\DevTools\MongoDB\Server\4.2\data\replicaset2" --replSet myreplicaset
	mongod --port 27018 --dbpath "C:\DevTools\MongoDB\Server\4.2\data\replicaset1" --replSet myreplicaset

   where "myreplicaset" is the name of replicaset.

3. Start using one of the servers(or, instance).
	mongo --port 27017

4. Initiate the replicaset.
	rs.initiate()

5. Add remaining servers to our replicaset.
	rs.add("localhost:27018")
	rs.add("localhost:27019")

6. You can check the status and configuration of your replicaset using
	rs.status()
	rs.conf()

7. For setting the priorities of servers use
	config = rs.conf()
	config.members[0].priority = 3 
	config.members[1].priority = 1
	config.members[2].priority = 2

8. Then reconfigure the replicaset
	rs.reconfig(config)


9. Now you can kill the primary server(replicaset0) and can see that the server which has the highest priority after the primary server would become the primary server.


NOTE :- All these commands were executed on git bash(Windows 10)
